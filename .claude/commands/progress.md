# /progress

セッション終了前に進捗状況を保存する。

## Goal

コンテキスト引き継ぎを確実に行い、次のセッションで即座に作業を再開できるようにする。

## Suggested Sequence

1. 現在のセッションで完了したタスクをリストアップする
2. 未完了タスクと残作業を整理する
3. 現在直面している問題・ブロッカーを記録する
4. 次セッションで最初にやるべきことを明記する
5. `docs/progress.md` に以下の形式で保存する：

```markdown
# 進捗ログ

## 最終更新: {日時}

## 完了済み
- [ ] タスク名

## 未完了 / 進行中
- [ ] タスク名（状況メモ）

## ブロッカー
- 問題の説明

## 次セッションの最初にやること
1. 優先タスク
```

6. `git commit` で進捗ファイルをコミットする

## Notes

- セッション終了前に必ず実行する
- 最低1時間に1回のコミットを推奨
