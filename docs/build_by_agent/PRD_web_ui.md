# PRD: Daru PDF Web UI（Web UI プロダクト要件定義書）

## 概要

Daru PDF Web UI は、PDF/画像ドキュメントの自動入力処理を可視化・管理するためのフロントエンドアプリケーションです。テキストベースのAPIレスポンスでは把握が困難なフィールドの状態、信頼度、問題点を視覚的に表示し、ユーザーが直感的に修正・承認できるインターフェースを提供します。

## 目的（Goals）

### 主要目的
1. **可視化**: フィールドの状態、信頼度、問題点を視覚的に表示
2. **効率化**: テキストベースの確認よりも迅速な問題発見と修正
3. **操作性**: 直感的なUIでフィールドの編集と承認を可能にする
4. **透明性**: 処理の進行状況と各ステージの状態をリアルタイムで表示

### 非目的（Non-Goals）
- PDFドキュメントの直接編集（座標調整はAPI経由）
- 複数ジョブの一括管理（初期版では単一ジョブ表示）
- ユーザー認証・権限管理（初期版では未実装）
- オフライン機能（常にAPIサーバーへの接続が必要）

## 想定ユーザー

### プライマリーユーザー
- **オペレーション担当者**: ドキュメント処理の結果を確認・修正する担当者
- **品質管理者**: 抽出精度を検証し、問題のあるフィールドを特定する管理者

### セカンダリーユーザー
- **開発者**: システムの動作確認とデバッグ
- **テスター**: 機能テストとUI/UX検証

## 機能要件

### 1. ジョブ入力・認証（JobInput）

#### 1.1 ジョブID入力
- **機能**: ユーザーがジョブIDを入力してジョブを読み込む
- **入力形式**: UUID形式の文字列（例: `dae6caff-e436-4643-a841-f2e0c5ba3ae1`）
- **バリデーション**: 
  - 空文字列の拒否
  - API接続確認後のみ送信可能
- **エラーハンドリング**:
  - ジョブが見つからない場合の明確なエラーメッセージ
  - APIサーバー接続不可の場合の警告表示
  - インメモリジョブの消失についての説明

#### 1.2 API接続確認
- **機能**: アプリ起動時にAPIサーバーの稼働状況を確認
- **エンドポイント**: `GET /health`
- **表示**:
  - 接続成功: 緑色の成功メッセージ
  - 接続失敗: 黄色の警告メッセージと解決方法の提示

#### 1.3 エラー表示
- **機能**: 詳細なエラーメッセージと解決方法の提示
- **表示内容**:
  - エラーの種類（ジョブ未検出、API接続不可など）
  - 一般的な原因のリスト
  - 解決方法の提案

### 2. ジョブビューア（JobViewer）

#### 2.1 ジョブヘッダー
- **表示項目**:
  - ジョブID（短縮表示: 最初の8文字 + "..."）
  - ジョブステータス（色分け表示）
    - `done`: 緑色
    - `running`: 青色
    - `awaiting_input`: 黄色
    - `blocked`: 赤色
  - 進捗率（パーセンテージ表示）
  - 現在のステージ名

#### 2.2 アクションボタン
- **ステータス別の表示**:
  - `awaiting_input`: 
    - "Submit Answers (N)" - 修正したフィールド数を表示
    - "Continue" - 処理を続行
  - `running`: 
    - "Running..." - 無効化されたボタン
  - `done`: 
    - "Download Output PDF" - 出力PDFをダウンロード
  - `next_actions`に"run"が含まれる場合:
    - "Run" - ジョブを実行

#### 2.3 自動更新
- **機能**: ジョブが`running`または`awaiting_input`状態の時、2秒間隔で自動更新
- **実装**: `useEffect`フックと`setInterval`を使用
- **クリーンアップ**: コンポーネントのアンマウント時にインターバルをクリア

#### 2.4 エラーバナー
- **機能**: API呼び出しエラーを画面上部に表示
- **スタイル**: 赤色背景、明確なエラーメッセージ

### 3. フィールドリスト（FieldList）

#### 3.1 フィールド表示
各フィールドに以下の情報を表示:
- **基本情報**:
  - フィールド名
  - フィールドタイプ（text, number, date, checkbox, radio, signature）
  - 必須フラグ（Requiredバッジ）
- **値情報**:
  - 現在の値（または"No value"）
  - 編集可能な場合は"Edit"ボタン
- **信頼度**:
  - パーセンテージ表示（0-100%）
  - 色分け:
    - 高信頼（≥80%）: 緑色
    - 中信頼（50-79%）: 黄色
    - 低信頼（<50%）: 赤色
    - 未設定: グレー
- **位置情報**:
  - ページ番号
  - 座標（x, y）

#### 3.2 フィルタリング
- **フィルタオプション**:
  - "All Fields": すべてのフィールド
  - "With Issues": 問題があるフィールドのみ
  - "Low Confidence": 信頼度が低い（<70%）フィールドのみ
  - "Missing Values": 値が未設定のフィールドのみ
- **実装**: ドロップダウンメニューで選択

#### 3.3 ソート
- **ソートオプション**:
  - "Sort by Name": フィールド名のアルファベット順
  - "Sort by Confidence": 信頼度の降順
  - "Sort by Page": ページ番号の昇順
- **実装**: ドロップダウンメニューで選択

#### 3.4 視覚的ハイライト
- **問題のあるフィールド**:
  - 左側に赤いボーダー（4px）
  - 背景色の変更（問題の種類に応じて）
- **低信頼度フィールド**:
  - 黄色の背景色（`#fef3c7`）
- **値が未設定のフィールド**:
  - 赤みがかった背景色（`#fee2e2`）

#### 3.5 インライン編集
- **機能**: フィールドの値を直接編集
- **操作フロー**:
  1. "Edit"ボタンをクリック
  2. テキスト入力フィールドが表示
  3. 値を編集
  4. "Save"で保存、"Cancel"でキャンセル
- **保存**: 編集した値は`answers`マップに保存され、一括送信可能

#### 3.6 問題表示
- **表示内容**:
  - 問題の種類（issue_type）
  - メッセージ
  - 深刻度アイコン（❌ エラー、⚠️ 警告）
  - 推奨アクション（suggested_action）
- **スタイル**:
  - エラー: 赤色背景（`#fee2e2`）
  - 警告: 黄色背景（`#fef3c7`）
  - 情報: 青色背景（`#dbeafe`）

### 4. サイドバー

#### 4.1 ソースフィールド表示
- **機能**: ソースドキュメントから抽出されたフィールドを表示
- **表示項目**:
  - フィールド名
  - 抽出された値
- **スタイル**: グレー背景のカード形式

#### 4.2 問題サマリー
- **表示項目**:
  - 総問題数
  - エラー数
  - 警告数
- **スタイル**: シンプルなリスト表示

#### 4.3 信頼度サマリー
- **機能**: すべてのターゲットフィールドの信頼度を一覧表示
- **表示形式**: 
  - フィールド名と信頼度のペア
  - 信頼度は色分け表示（緑/黄/赤）
- **スタイル**: 2カラムレイアウト（フィールド名 | 信頼度）

### 5. API統合（jobClient）

#### 5.1 API関数
- `getJob(jobId)`: ジョブ情報の取得
- `runJob(jobId, request)`: ジョブの実行
- `submitAnswers(jobId, answers)`: フィールド値の送信
- `getPagePreview(documentId, page)`: ページプレビューの取得（将来実装）
- `downloadOutput(jobId)`: 出力PDFのダウンロード
- `checkApiHealth()`: API接続確認

#### 5.2 エラーハンドリング
- **HTTPエラー**: ステータスコードに基づくエラーメッセージ
- **APIエラー**: JSONレスポンスの`error`フィールドからメッセージを抽出
- **ネットワークエラー**: 接続不可の場合の明確なメッセージ

#### 5.3 型安全性
- **TypeScript型定義**: すべてのAPIレスポンスに型定義
- **インターフェース**: 
  - `JobContext`
  - `Field`
  - `Issue`
  - `Evidence`
  - その他必要な型

## 非機能要件

### パフォーマンス
- **初期ロード時間**: 2秒以内
- **API呼び出し**: タイムアウト30秒
- **自動更新**: 2秒間隔（サーバー負荷を考慮）
- **レンダリング**: 100フィールドまでスムーズにスクロール可能

### レスポンシブデザイン
- **デスクトップ**: 1280px以上で最適表示
- **タブレット**: 768px以上で基本機能を維持
- **モバイル**: 将来的な対応（現時点では未対応）

### ブラウザサポート
- **必須**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **機能**: ES2020+、Fetch API、CSS Grid/Flexbox

### アクセシビリティ
- **キーボード操作**: すべての操作がキーボードで可能
- **スクリーンリーダー**: 基本的なARIA属性の使用
- **色のコントラスト**: WCAG AA準拠

### セキュリティ
- **CORS**: APIサーバー側で適切に設定
- **XSS対策**: Reactの自動エスケープに依存
- **認証**: 将来的にJWTトークンを使用（現時点では未実装）

## UI/UX仕様

### デザインシステム

#### カラーパレット
- **プライマリ**: `#3b82f6` (青)
- **成功**: `#22c55e` (緑)
- **警告**: `#eab308` (黄)
- **エラー**: `#ef4444` (赤)
- **背景**: `#f9fafb` (ライトグレー)
- **テキスト**: `#111827` (ダークグレー)

#### タイポグラフィ
- **見出し**: 18-28px、font-weight: 600-700
- **本文**: 14px、font-weight: 400
- **キャプション**: 12px、font-weight: 400
- **フォント**: システムフォントスタック

#### スペーシング
- **コンテナパディング**: 16-24px
- **要素間隔**: 8pxの倍数（8, 16, 24, 32px）
- **ボーダー半径**: 4-8px

#### コンポーネントスタイル
- **ボタン**: 
  - プライマリ: 青背景、白文字
  - セカンダリ: 白背景、グレーボーダー
  - ホバー: 色の濃淡変化
  - 無効: 50%透明度
- **入力フィールド**: 
  - 白背景、グレーボーダー
  - フォーカス: 青ボーダー
- **カード**: 
  - 白背景、影付き
  - 角丸

### ユーザーフロー

#### フロー1: ジョブの確認と修正
1. ジョブIDを入力
2. ジョブ情報が読み込まれる
3. フィールドリストで問題のあるフィールドを確認
4. フィルタで"With Issues"を選択して問題フィールドに絞り込み
5. 問題のあるフィールドをクリックして詳細確認
6. "Edit"ボタンで値を修正
7. "Submit Answers"で修正を送信
8. ジョブが再実行される
9. 結果を確認

#### フロー2: 低信頼度フィールドの確認
1. ジョブを読み込む
2. フィルタで"Low Confidence"を選択
3. 信頼度サマリーで各フィールドの信頼度を確認
4. 低信頼度フィールドの値を検証
5. 必要に応じて修正して送信

#### フロー3: 出力PDFのダウンロード
1. ジョブが`done`状態になるまで待機
2. "Download Output PDF"ボタンをクリック
3. PDFファイルがダウンロードされる

## リスクと対策

### リスク1: APIサーバーが応答しない
- **影響**: アプリケーションが使用不可
- **対策**: 
  - 接続確認機能の実装
  - 明確なエラーメッセージ
  - リトライ機能（将来実装）

### リスク2: 大量のフィールドでパフォーマンス低下
- **影響**: UIの応答性低下
- **対策**: 
  - 仮想スクロールの実装（将来）
  - ページネーション（将来）
  - 現在は100フィールドまで最適化

### リスク3: ジョブがインメモリで消失
- **影響**: ユーザーの作業が失われる
- **対策**: 
  - 明確な警告メッセージ
  - 将来はデータベース永続化
